## 1. このファイルは何をするものか

このサイトの一番の特徴である「AIが寄り道スポットを考えてくれる」部分の、裏側の処理を作るのが担当Cの仕事です。

具体的には、以下のような入出力を行う関数を1つ作ります。

- **入力**：現在地（例：`"新宿駅"`）と目的地（例：`"浅草""`）という2つの文字列
- **中でやること**：AI（人工知能のサービス）に「この2つの間で寄り道できる場所を教えて」と問い合わせる
- **出力**：[00_overview.md](./00_overview.md)で決めた形の配列（`[{name, description, reason}, ...]`）

**注意：ここでは画面の見た目や、入力フォームは作りません。** それは担当A・Bの仕事です。担当Cは「AIとやり取りして、決まった形のデータに変換して返す」ところまでが担当範囲です。

## 3. 事前に知っておく言葉のおさらい

- **API（エーピーアイ）**：あるプログラムが、別のプログラム（今回はAIのサービス）に対して「これをやってほしい」とお願いするための窓口です。多くの場合、インターネット経由でお願いを送り、返事を受け取ります。
- **プロンプト**：AIに対して送る「お願いの文章」のことです。「〇〇について3つ提案して」のような文章がこれにあたります。
- **非同期処理（async/await）**：AIへの問い合わせはすぐには終わらず、数秒程度の時間がかかります。「時間がかかる処理の結果を待ってから、次の処理に進む」という書き方が`async`（この関数は時間がかかる処理を含みますという宣言）と`await`（ここで結果を待つ）です。
- **JSON（ジェイソン）**：データを文字列として表現するときの、よく使われる決まった書き方です。例：`{"name": "上野公園"}`。AIにも「この書き方で返事をしてください」とお願いできます。
- **JSON.parse()**：JSONの形式で書かれた文字列を、JavaScriptで実際に使えるオブジェクトや配列に変換するための関数です。
- **try/catch**：「失敗するかもしれない処理」を`try`の中に書き、もし失敗したら`catch`の中の処理が代わりに実行される、というエラー対策の書き方です。

## 5. 手順

### ステップ1：関数の入り口だけ先に作る（ダミー版）

本物のAIとの接続には時間がかかることがあります。担当A・B・Dを待たせないために、まずは中身が仮のデータでもよいので、「決められた形」を返す関数を用意しましょう。

```js
export async function getSpots(from, to) {
  // ダミー版：まずはこれで担当Dに渡して結合テストしてもらう
  return [
    { name: "上野公園", description: "自然を楽しみながら休憩できる", reason: "ルート上で気分転換しやすいため" },
    { name: "秋葉原", description: "買い物や観光に立ち寄りやすい", reason: "乗り換え駅から近いため" },
  ];
}
```

- `export`：他のファイル（`App.jsx`）からこの関数を読み込めるようにするための宣言です。
- `async function getSpots(from, to) {`：`getSpots`という名前の、時間がかかる処理を含む関数です。`from`（現在地）と`to`（目的地）を受け取ります。今はまだ使っていませんが、あとで使います。
- `return [ ... ]`：[00_overview.md](./00_overview.md)で決めた形の配列を、そのまま返しています。

この時点で担当Dに「ダミー版ができた」と共有し、結合を先に進めてもらってください。

### ステップ2：AIに送る文章（プロンプト）を組み立てる

現在地・目的地を文章の中に埋め込んで、AIに「この形式で返して」と明確に指定する文章（プロンプト）を作ります。

```js
function buildPrompt(from, to) {
  return `
${from}から${to}まで移動する途中で立ち寄れる「寄り道スポット」を3つ提案してください。
以下のJSON配列の形式だけで出力してください。説明文や前置きは不要です。

[
  { "name": "スポット名", "description": "どんな場所か", "reason": "おすすめ理由" }
]
`;
}
```

- `` ` `` （バッククォート）で囲む書き方をテンプレートリテラルといい、`${from}`のように書くと、その場所に変数の中身がそのまま埋め込まれます。
- なぜJSON形式を指定するのか：AIは基本的に自由な文章で答えるため、何も指定しないと「上野公園がおすすめです。理由は〜」のような、プログラムでは扱いにくい文章で返ってきます。あらかじめ「この形式で返して」と具体的な見本を見せることで、プログラムが読み取りやすい形で返ってきやすくなります。

### ステップ3：AI APIを実際に呼び出す

チームで選んだAIサービスの手順（SDKの使い方、または`fetch`というインターネット通信の仕組み）に沿って、ステップ2で作ったプロンプトを送信します。ここは使用するAIサービスによって書き方が変わるため、公式ドキュメントを参照しながら進めてください。

```js
export async function getSpots(from, to) {
  const prompt = buildPrompt(from, to);

  // ここでAI APIを呼び出す（使用するSDK/fetchに置き換える）
  const responseText = await callAI(prompt); // callAIは呼び出すAI次第で実装が変わる部分

  return parseSpots(responseText);
}
```

- `const prompt = buildPrompt(from, to);`：ステップ2で作った関数を呼び出し、実際の問い合わせ文を作ります。
- `await callAI(prompt);`：AIへの問い合わせ（時間がかかる処理）が終わるのを待ってから、その返事（`responseText`という文字列）を受け取ります。`callAI`という関数名は仮の名前です。実際には、使用するAIサービスの呼び出し方法に置き換えてください。
- `return parseSpots(responseText);`：AIからの生の返事（文字列）を、次のステップで扱いやすい形に変換してから返します。

### ステップ4：AIの返事をJSONとして読み取る

AIの返事（文字列）を、実際にプログラムで使える配列に変換します。

```js
function parseSpots(responseText) {
  try {
    const spots = JSON.parse(responseText);
    return spots;
  } catch (e) {
    console.error("AIの返事をJSONとして読み取れませんでした:", responseText);
    return []; // 失敗したときは空配列を返し、画面側でエラー表示させる
  }
}
```

- `JSON.parse(responseText)`：JSON形式の文字列を、JavaScriptのオブジェクト・配列に変換します。これが成功すれば、`spots`は`[{name: ..., description: ..., reason: ...}, ...]`という、そのまま使える配列になります。
- `try { ... } catch (e) { ... }`：`JSON.parse`は、渡された文字列がJSONとして正しくない場合にエラーを起こします。`try`の中で試してみて、失敗したら`catch`の中の処理（エラーの記録と、空配列を返す）に切り替えます。
- なぜ失敗することがあるのか：AIは指定したはずのJSON形式を守らず、前後に余計な文章をつけて返してくることがあります。うまく変換できない場合は、プロンプトで「JSON以外は絶対に出力しないでください」のように念を押す、または返ってきた文字列から余計な部分を取り除く処理を追加する、といった対策が必要になることがあります。

### ステップ5：エラー処理を仕上げる

APIキーが間違っている、インターネット接続に失敗した、といったケースにも備えます。AIへの問い合わせ自体が失敗する可能性があるため、その部分も`try/catch`で囲んでおくと安全です。

```js
export async function getSpots(from, to) {
  try {
    const prompt = buildPrompt(from, to);
    const responseText = await callAI(prompt);
    return parseSpots(responseText);
  } catch (e) {
    console.error("AIへの問い合わせに失敗しました:", e);
    throw e; // 呼び出し元（App.jsx）にエラーを伝える
  }
}
```

- `throw e;`：ここでエラーをそのまま担当D（`App.jsx`）に伝え直します。担当Dの`handleSearch`関数が、この`throw`されたエラーを`catch`で受け止めて、画面に「取得に失敗しました」のようなメッセージを出す仕組みになっています（詳しくは[04_D_app.md](./04_D_app.md)参照）。

## 6. つまずきやすいポイント

- **AIの返事が期待通りのJSONにならない**：プロンプトの指定が曖昧だと起こりやすいです。ステップ2の見本をもっと具体的にする、「説明文は一切書かないでください」のように念押しを増やすなど、プロンプトを調整してみてください。
- **APIキーのエラーが出る**：`.env`ファイルの書き方や読み込み方が正しいか確認してください。キーの前後に余計なスペースや改行が入っていないかもよくある原因です。
- **`await`をつけ忘れて、期待した値が取れない**：`async`な関数（時間がかかる処理をする関数）の結果を受け取るときは、必ず`await`をつけてください。つけ忘れると、実際の値ではなく「まだ処理中です」という状態のオブジェクト（Promise）が返ってきてしまいます。
