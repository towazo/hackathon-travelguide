# 担当B：結果表示 手順書

担当ファイル：`src/components/SpotResultList.jsx`、`src/components/SpotCard.jsx`、`src/styles/App.css`

> このページだけを読んでも作業できるように書いていますが、`spots`や`props`といった言葉が分からない場合は、先に [00_overview.md](./00_overview.md) を読んでください。特に「4. 用語集」「5. データの形」は必ず目を通してください。

---

## 1. このファイルは何をするものか

AIが提案してきた「寄り道スポットの一覧」を、ユーザーが見やすいカードのような形で画面に並べるのが、担当Bの仕事です。

具体的には、以下のような見た目を作ります。

```
┌─────────────────────────────┐
│ 上野公園                            │
│ 自然を楽しみながら休憩できる          │
│ おすすめ理由：移動ルートの近くにあり… │
└─────────────────────────────┘
┌─────────────────────────────┐
│ 秋葉原                              │
│ 買い物や観光に立ち寄りやすい          │
│ おすすめ理由：乗り換え駅から近いため   │
└─────────────────────────────┘
```

このカードが1件のスポットにつき1つずつ並びます。何件表示するかはAIの返事次第で変わるため、「決まった数だけ表示する」のではなく、「渡された数だけ表示する」ように作ります。

**注意：ここでAIに問い合わせる処理や、入力フォームは作りません。** それは担当C・Aの仕事です。担当Bは「渡されたデータを表示する」ところまでが担当範囲です。

---

## 2. 全体の中でのこのファイルの位置

[00_overview.md](./00_overview.md)の流れ図の、一番下の部分がこのファイルの担当です。

```
┌─────────────────────────┐
│ App（担当D）                  │
│ ・spots にAIの返事を保存する         │
└─────────────────────────┘
    ↓ spots を渡す
┌─────────────────────────┐
│ SpotResultList（★担当B）      │
│ ・spots を1件ずつ取り出す           │
└─────────────────────────┘
    ↓ spot（1件分）を渡す
┌─────────────────────────┐
│ SpotCard（★担当B）           │
│ ・名前・説明・理由を画面に表示          │
└─────────────────────────┘
```

`SpotResultList`は「一覧をどう並べるか」、`SpotCard`は「1件をどう見せるか」を分担しています。1つのファイルに全部書かず、2つに分ける理由は、「1件分の見た目を変えたいときは`SpotCard`だけ触ればよい」「並べ方（グリッドにする、横並びにするなど）を変えたいときは`SpotResultList`だけ触ればよい」というように、変更する場所を分かりやすくするためです。

---

## 3. 事前に知っておく言葉のおさらい

- **props**：親から子のコンポーネントに渡されるデータのことです。`SpotResultList`は`App`から`spots`（配列）を、`SpotCard`は`SpotResultList`から`spot`（1件分のオブジェクト）を受け取ります。
- **配列（array）**：複数のデータを順番に並べて持っておく入れ物です。`[要素1, 要素2, 要素3]`のように書きます。
- **`.map()`**：配列の中身を1つずつ取り出して、別の形に変換するための関数です。今回は「スポットのデータ1件」を「そのスポットを表示するカード（`SpotCard`）」に変換するために使います。
- **key（キー）**：`.map()`で複数の部品を作るとき、Reactが「どの部品がどれか」を区別するために必要な目印です。

## 4. 手順

### ステップ1：まずダミーデータで作業を始める

担当C（AI連携）の実装がまだ終わっていなくても、担当Bは作業を始められます。[00_overview.md](./00_overview.md)の「5. データの形」に載っている、以下のような仮のデータを使って、先に見た目を作ってしまいましょう。

```js
const dummySpots = [
  { name: "上野公園", description: "自然を楽しみながら休憩できる", reason: "移動ルートの近くにあり気分転換になるため" },
  { name: "秋葉原", description: "買い物や観光に立ち寄りやすい", reason: "乗り換え駅から近いため" },
];
```

このダミーデータは、あとで本物のAIの返事に差し替わるだけなので、今は「こういう形のデータが来る」という前提で作業を進めれば問題ありません。

### ステップ2：SpotCard.jsx を作る（1件分の表示）

先に「小さい部品」から作ると、全体が分かりやすくなります。1件分のスポット情報（`spot`）を受け取り、名前・説明・理由を表示するだけのシンプルな部品です。

```js
function SpotCard({ spot }) {
  return (
    <div className="spot-card">
      <h3>{spot.name}</h3>
      <p>{spot.description}</p>
      <p className="reason">おすすめ理由：{spot.reason}</p>
    </div>
  );
}

export default SpotCard;
```

1行ずつ説明します。

- `function SpotCard({ spot }) {`：`SpotCard`という部品は、`spot`という名前のprops（1件分のスポットのオブジェクト）を受け取ります。
- `<h3>{spot.name}</h3>`：`spot`オブジェクトの中の`name`という項目を、見出しとして表示します。`{ }`で囲むことで、JSXの中にJavaScriptの値を埋め込めます。
- 同じように`description`と`reason`を表示します。
- `className="spot-card"`：あとでCSS（見た目のデザイン）を当てるための目印です。HTMLでいう`class`属性と同じ役割です。

**注意：`spot`は1件分のオブジェクトであって、配列ではありません。** `spot.name`のように、`.`（ドット）でオブジェクトの中の項目を取り出します。

### ステップ3：SpotResultList.jsx を作る（一覧表示）

`spots`（複数件の配列）を受け取り、`.map()`を使って1件ずつ`SpotCard`に渡します。

```js
import SpotCard from "./SpotCard";

function SpotResultList({ spots }) {
  if (spots.length === 0) {
    return <p>寄り道スポットはまだありません。</p>;
  }

  return (
    <div className="spot-result-list">
      {spots.map((spot) => (
        <SpotCard key={spot.name} spot={spot} />
      ))}
    </div>
  );
}

export default SpotResultList;
```

1行ずつ説明します。

- `import SpotCard from "./SpotCard";`：さきほど作った`SpotCard`という部品を、このファイルで使えるように読み込みます。
- `function SpotResultList({ spots }) {`：`spots`という名前のprops（配列）を受け取ります。
- `if (spots.length === 0) { return <p>...</p>; }`：まだ何も検索していない、またはAIが何も見つけられなかった場合（配列が空のとき）は、カードの代わりに案内文を表示します。`spots.length`は配列の件数です。
- `{spots.map((spot) => ( <SpotCard key={spot.name} spot={spot} /> ))}`：`spots`の中身を1件ずつ取り出し（取り出した1件を`spot`という名前で扱う）、それぞれについて`SpotCard`を1つ作ります。`key={spot.name}`は、Reactに「これらは別々のカードですよ」と伝えるための目印です（今回はスポット名が重複しない前提でOKです）。

### ステップ4：App.css で最低限の見た目を整える

凝ったデザインは不要です。以下のような最低限のクラス（`className`で指定した名前に対応するデザインのルール）があれば十分です。

```css
.spot-result-list {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.spot-card {
  border: 1px solid #ccc;
  border-radius: 8px;
  padding: 12px;
}

.reason {
  color: #666;
  font-size: 0.9em;
}
```

- `.spot-result-list`：カードを縦に並べ（`flex-direction: column`）、カード同士の間に12pxの隙間（`gap`）を空けます。
- `.spot-card`：1枚のカードに、薄い枠線・角丸・内側の余白をつけます。
- `.reason`：おすすめ理由の文字を少し薄いグレーにして、他の文章と見分けやすくします。

### ステップ5：動作確認

担当Aが作った入力フォームや、担当Dの結合作業を待たずに、自分だけで見た目を確認したい場合は、一時的に`App.jsx`を書き換えず、`SpotResultList`を直接試すファイル（例えば`src/main.jsx`を一時的に書き換えるなど）でステップ1のダミーデータを渡して表示してみるとよいでしょう。ただし本来の結合は担当Dが行うため、確認が終わったら元に戻してください。

すでに担当Dの結合ができている場合は、実際にブラウザで画面を開き、カードが複数枚、崩れずに並んで表示されるか確認してください。

## 5. つまずきやすいポイント

- **`Cannot read properties of undefined`のようなエラーが出る**：`spot.name`のように書いているのに、`spot`自体が`undefined`（存在しない）場合に起きます。`SpotResultList`側で`spot={spot}`のように、propsをちゃんと渡しているか確認してください。
- **カードが1枚も表示されない**：`spots`が空配列`[]`のままになっていないか確認してください（担当Dがまだ`spots`を更新していない可能性があります）。
- **`key`を指定していないという警告が出る**：`.map()`で複数の部品を作るときは、必ず`key`を指定する必要があります。今回は`spot.name`を`key`に使っていますが、もし名前が重複する可能性がある場合は、他の一意な値を使うよう相談してください。
