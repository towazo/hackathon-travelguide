## 1. このファイルは何をするものか

このサイトを開くと、ユーザーは最初に「現在地」と「目的地」を入力する画面を見ます。その入力欄とボタンを作るのが、担当Aの仕事です。

具体的には、以下のような見た目と動作を作ります。

```
┌─────────────────────────────┐
│ 現在地：[ 新宿駅            ]     │
│ 目的地：[ 浅草              ]     │
│                                    │
│           [ 検索する ]            │
└─────────────────────────────┘
```

ユーザーがこの2つの入力欄に文字を打ち込み、「検索する」ボタンを押すと、その2つの文字（現在地・目的地）を、担当D（`App.jsx`）に伝えます。

**注意：ここでAIに問い合わせたり、結果を表示したりする処理は書きません。** それは担当C・Bの仕事です。担当Aは「入力を受け取って、親に伝える」ところまでが担当範囲です。

## 2. 全体の中でのこのファイルの位置

[00_overview.md](./00_overview.md)の流れ図の、一番上の部分がこのファイルの担当です。

```
【ユーザーの操作】
    ↓ 「新宿駅」「浅草」と入力してボタンを押す
┌─────────────────────────┐
│ SpotSearchForm（★ここが担当A）  │
└─────────────────────────┘
    ↓ onSearch("新宿駅", "浅草") を呼び出す
┌─────────────────────────┐
│ App（担当D）                  │
└─────────────────────────┘
```

`onSearch`という関数は、担当Dがあらかじめ用意して、担当Aのコンポーネントに「props」として渡してくれます。担当Aは、その関数を「入力が完了したタイミングで呼び出す」だけでよく、関数の中身がどうなっているかを知る必要はありません。これがコンポーネントを分けるメリットの1つです。

## 3. 事前に知っておく言葉のおさらい

- **useState**：入力中の文字のように「今どんな値になっているか」を覚えておくための仕組みです。値が変わると、画面が自動的に描き直されます。
- **props**：親（`App.jsx`）から渡されるデータや関数のことです。今回は`onSearch`という関数が渡されてきます。
- **イベント（onChange, onSubmit）**：`onChange`は入力欄の文字が変わった瞬間、`onSubmit`はフォームが送信された瞬間に呼ばれる仕組みです。

## 4. 手順

### ステップ1：ファイルの土台と、入力値を覚える場所を作る

入力欄に文字を打つたびに、その値をどこかに覚えておく必要があります。Reactでは`useState`という仕組みを使います。

```js
import { useState } from "react";

function SpotSearchForm({ onSearch }) {
  const [from, setFrom] = useState(""); // 現在地の入力値
  const [to, setTo] = useState("");     // 目的地の入力値

  // この続きをステップ2以降で書いていく
}

export default SpotSearchForm;
```

1行ずつ説明します。

- `import { useState } from "react";`：Reactが用意している「値を覚えておく道具」を使えるようにする宣言です。
- `function SpotSearchForm({ onSearch }) {`：この部品（コンポーネント）の名前は`SpotSearchForm`です。`{ onSearch }`の部分で、親から渡されてくる`onSearch`という関数を受け取っています。
- `const [from, setFrom] = useState("");`：`from`は「今の現在地の入力値」、`setFrom`は「その値を書き換えるための専用の関数」です。最初は空文字`""`から始まります。
- 同じように`to`と`setTo`を目的地用に用意します。

**なぜ2つの`useState`が必要なのか**：現在地と目的地は別々の入力欄なので、それぞれ別々に値を覚えておく必要があるからです。

### ステップ2：入力欄（JSX）を作る

`function`の中の、まだ何も書いていない部分に、画面に表示する内容（JSX）を`return`で書きます。

- 「現在地」用の`<input>`タグを1つ
- 「目的地」用の`<input>`タグを1つ
- 送信用の`<button>`タグを1つ

`<input>`タグには、以下の3つを指定します。

- `value={from}`：今の入力値を、さきほど作った`from`にする（＝入力欄の中身とstateを連動させる）
- `onChange={(event) => setFrom(event.target.value)}`：文字が変わるたびに、新しい文字（`event.target.value`）で`from`を書き換える
- `placeholder="現在地（例：新宿駅）"`：何も入力されていないときに薄く表示されるヒント文言

**`event.target.value`とは**：入力欄で今まさに入力された文字列そのものが入っています。「入力欄で何かが変わった」というイベント（きっかけ）の詳細情報の中に、「変わった後の値」が入っている、とイメージしてください。

目的地用の`<input>`も同じ考え方で、`to`と`setTo`を使って作ります。

### ステップ3：送信処理を作る

ボタンが押されたときに実行される関数を用意し、その中で親から受け取った`onSearch`を呼び出します。

```js
function handleSubmit(event) {
  event.preventDefault(); // ページが再読み込みされるのを防ぐ
  if (!from || !to) return; // どちらかが空欄なら何もしない
  onSearch(from, to); // 親（App.jsx）に現在地・目的地を伝える
}
```

- `event.preventDefault()`：`<form>`タグを使う場合、本来ボタンを押すとページが再読み込みされてしまいます。それを止めるためのおまじないです。
- `if (!from || !to) return;`：`from`が空文字（つまり`""`）のときは`!from`が`true`になります。どちらかが空なら、それ以上処理を進めずに関数を終了します。
- `onSearch(from, to);`：ここが一番重要な行です。親から渡された`onSearch`という関数を、今の`from`・`to`の値を渡して実行します。これにより、「入力された値」が担当Dの`App.jsx`に伝わります。

この`handleSubmit`関数を、`<form onSubmit={handleSubmit}>`のように紐づけます。ボタンは`<button type="submit">検索する</button>`のように、フォームの送信ボタンとして置くと分かりやすいです。

### ステップ4：動作確認（担当D・Cの完成を待たずに確認する方法）

担当Dの`onSearch`がまだ完成していない場合でも、自分の入力欄が正しく動いているかは以下の方法で確認できます。

```js
// 動作確認用（あとで元に戻してOK）
function handleSubmit(event) {
  event.preventDefault();
  console.log("入力値の確認:", from, to);
}
```

ブラウザで画面を開き、キーボードの`F12`（またはブラウザのメニューから「開発者ツール」）を開いて「Console（コンソール）」タブを見てください。入力した文字を打ち込んでボタンを押し、コンソールに`入力値の確認: 新宿駅 浅草`のように表示されればOKです。

確認できたら、`console.log(...)`の行を消して、`onSearch(from, to);`に戻してください。

## 5. つまずきやすいポイント

- **入力欄に文字が打てない／打っても表示されない**：`value={from}`は指定したのに`onChange`を指定し忘れると、入力欄の中身が固定されて文字が打てなくなります。`value`と`onChange`は必ずセットで指定してください。
- **ボタンを押しても何も起きない**：`<button>`の`type`が指定されていない、または`type="button"`になっていると、`<form>`の`onSubmit`が反応しないことがあります。フォーム送信用のボタンには`type="submit"`を指定してください。
- **`onSearch is not a function`のようなエラーが出る**：担当Dの`App.jsx`側で、まだ`onSearch`というpropsを渡していない可能性があります。担当Dに確認してください。
